# 首页运营配置后台

为运营人员提供即时、灵活的首页内容管理能力。

- **兜底逻辑**: 如某展位无在线素材，则该区域高度自动收起 (对于副 Banner) 或显示默认占位图 (对于主 Banner)。

## 2. 金刚位配置 (King Kong Area)

首页 Banner 下方的 5 个核心功能入口。

### 配置能力
支持运营动态调整金刚位的名称、图标和跳转目标，以适应不同季节的运营重点 (e.g., 雪季推"购票"，夏季推"滑雪营")。

### 配置列表
| 排序 | 图标 (Icon) | 名称 (Title) | 跳转目标 (Link) | 角标 (Badge) | 状态 |
|--|---|---|---|---|---|
| 1 | [图片] | 购雪票 | /pages/ticket/home | [🔥热门] | ✅ |
| 2 | [图片] | 赚钱币 | /pages/task/index | [空] | ✅ |
| ... | ... | ... | ... | ... | ... |

> **限制**: 建议保持 5 个在线入口以维持 UI 布局平衡。

## 3. 热门雪票配置

首页"热门推荐"栏目的运营干预。

### 选品逻辑
采用 **"纯人工配置"** 模式。
需运营人员手动配置产品图片和运营标签，以确保首页展示的视觉统一性。

### 配置项
1. **列表管理**:
   - **新增**: 选择产品库中的商品添加至列表。
   - **排序**: 拖拽调整展示顺序。
   - **移除**: 将商品从推荐列表删除。

2. **内容编辑 (必填)**:
   - **展示图片**: 需单独上传符合首页尺寸规范的图片 (区别于详情页轮播图)。
   - **运营标签**: 输入 2-4 字的营销标签 (e.g., "滑雪大神", "新手友好")。
   - **推荐理由**: (可选) 一句话推荐语。

3. **展示规则**:
   - 严格按照后台配置列表顺序展示。
   - 哪怕配置了 100 个，前端也只取前 10 个 (或按 UI 约定数量) 展示。
   - 若某商品下架，该配置项自动失效不展示。

---

## 4. 异常处理与缓存

- **缓存策略**: APP 启动时拉取最新配置，缓存 1 小时。
- **数据容错**: 若配置的跳转商品已下架，前端自动过滤该条目，不予展示。
- **图片压缩**: 上传后自动生成 WebP 格式供移动端加载。
