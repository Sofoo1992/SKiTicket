# **滑雪社交App \- 活动板块 PRD (MVP版)**

> **版本说明**：本文档为MVP（最小可行产品）版本，聚焦核心功能快速验证市场需求。

---

## **一、 全局类目架构 (Taxonomy)**

### **1.1 MVP活动类型（8种）**

> **分类逻辑**：按活动内容分类，不按发起方分类。通过"发起方角色"和"活动标签"来体现官方性/俱乐部属性。

发布入口采用 **8宫格** 布局。

| ID | 一级主类目 | 二级细分场景 | 核心特征 (Key Feature) | 适用场景 |
| :---- | :---- | :---- | :---- | :---- |
| **01** | **🏂 约滑** | AA搭子、组队约滑、品牌试滑 | **水平敏感**。核心是"滑行等级"和"板型"。 | 普通用户、俱乐部、官方均可发起 |
| **02** | **📚 教学** | 滑雪教学、技术指导 | **资质敏感**。核心是"教练认证"和"教学内容"。 | 认证教练、俱乐部教练 |
| **03** | **🚗 拼车/出行** | 车找人、人找车 | **空间敏感**。核心是"板包容量"和"路线"。 | 普通用户、俱乐部均可发起 |
| **04** | **🏨 拼房/住宿** | 酒店拼房、民宿合租 | **生活习惯敏感**。核心是"性别"和"呼噜"。 | 普通用户、俱乐部均可发起 |
| **05** | **🍻 吃喝玩乐** | 雪后聚餐、开板节、俱乐部年会 | **氛围敏感**。核心是"主题"和"人数"。 | 普通用户、俱乐部、官方均可发起 |
| **06** | **📷 摄影/跟拍** | 专业跟拍、互拍搭子 | **交付敏感**。核心是"设备"和"作品展示"。 | 认证摄影师、俱乐部签约摄影师 |
| **07** | **🏆 赛事/挑战** | 线上轨迹赛、俱乐部赛、官方赛 | **数据敏感**。核心是"轨迹排行榜"和"挑战目标"。 | 俱乐部、平台官方 |
| **08** | **🎫 官方团购** | 雪票拼团、住滑套餐 | **价格敏感**。核心是"折扣"和"限量"。 | 仅平台官方 |

**说明**：
- **开板节、品牌试滑、嘉年华** → 使用"🏂 约滑"或"🍻 吃喝玩乐"类型，通过活动标签体现
- **俱乐部年会、团建** → 使用"🍻 吃喝玩乐"类型，标注"俱乐部活动"标签
- **日程表、主办方介绍** → 在活动说明（富文本）中编辑，不需要单独的类型

## **二、 组织方权限体系 (Organizer Permission)**

### **2.1 角色定义（MVP支持5种角色）**

| 角色类型 | 准入条件 | 可发起活动范围 | 收费权限 | 对接现有系统 |
| :---- | :---- | :---- | :---- | :---- |
| **普通用户** | 完成实名认证 | 约滑、拼车、拼房、吃喝玩乐、互拍 | 仅免费/线下AA | 实名认证系统 |
| **认证教练** | 实名 + 教练资质审核通过 | 教学 + 普通用户全部权限 | 可线上收费（平台分账） | 认证教练体系✅ |
| **认证摄影师** | 实名 + 作品审核通过 | 摄影/跟拍 + 普通用户全部权限 | 可线上收费（平台分账） | 认证摄影师体系✅ |
| **俱乐部** | 营业执照 + 法人认证 | 全部8种活动类型 | 可线上收费（平台分账） | 俱乐部认证体系 |
| **平台官方** | 平台内部账号 | 全部8种活动类型 | 全部权限 | 内部账号管理 |

**MVP说明**：
- ✅ 对接现有的**认证教练体系**、**认证摄影师体系**
- ✅ 对接现有的**个人收款账户管理**、**分账规则**、**提现功能**
- ✅ 教练和摄影师的付费服务支持线上支付+自动分账
- ✅ 俱乐部作为机构账号，可发起大型活动、赛事、收费服务等

### **2.2 活动类型权限矩阵**

| 活动类型 | 普通用户 | 认证教练 | 认证摄影师 | 俱乐部 | 平台官方 | 支持的收费方式 |
| :---- | :----: | :----: | :----: | :----: | :----: | :---- |
| 🏂 **约滑** | ✅ | ✅ | ✅ | ✅ | ✅ | 免费 / 线下AA / 线上支付（俱乐部/官方） |
| 📚 **教学** | ❌ | ✅ | ❌ | ✅ | ✅ | 免费 / 线下AA / 线上支付（分账） |
| 🚗 **拼车/出行** | ✅ | ✅ | ✅ | ✅ | ✅ | 免费 / 线下AA |
| 🏨 **拼房/住宿** | ✅ | ✅ | ✅ | ✅ | ✅ | 免费 / 线下AA |
| 🍻 **吃喝玩乐** | ✅ | ✅ | ✅ | ✅ | ✅ | 免费 / 线下AA / 线上支付（俱乐部/官方） |
| 📷 **摄影/跟拍** | ✅（仅互拍） | ✅ | ✅ | ✅ | ✅ | 免费 / 线下AA / 线上支付（分账） |
| 🏆 **赛事/挑战** | ❌ | ❌ | ❌ | ✅ | ✅ | 免费 / 线上支付（分账/平台收款） |
| 🎫 **官方团购** | ❌ | ❌ | ❌ | ❌ | ✅ | 线上支付（平台收款） |

**核心规则**：
1. **普通用户**：可发起社交类活动（约滑、拼车、拼房、吃喝玩乐、互拍），只能免费或线下AA
2. **认证教练**：
   - 可发起**付费教学**，支持线上支付，平台自动分账（教练85%，平台15%）
   - 继承普通用户所有权限
3. **认证摄影师**：
   - 可发起**付费跟拍**，支持线上支付，平台自动分账（摄影师85%，平台15%）
   - 继承普通用户所有权限
   - 普通用户可发起"互拍搭子"（免费），但不能提供付费跟拍服务
4. **俱乐部**：
   - 可发起**所有8种活动类型**（包括赛事/挑战）
   - 可发起收费活动，平台自动分账（俱乐部85%，平台15%）
   - 典型场景：开板节、俱乐部年会、俱乐部赛事、俱乐部教学等
5. **平台官方**：可发起所有类型活动，包括官方团购（100%归平台）
6. **线下AA说明**：发起人标注预估费用（如"人均约¥80"），参与者报名时无需支付，活动现场AA结算

## **三、 费用类型与支付方式 (Payment Types)**

### **3.1 三种费用类型**

| 费用类型 | 适用场景 | 资金流向 | 报名流程 |
| :---- | :---- | :---- | :---- |
| **免费** | 纯社交组局，不涉及费用 | 无资金流转 | 直接点击报名成功 |
| **线下AA** | 拼车油费、聚餐饭钱等 | 线下现场AA，平台不涉及 | 直接报名，查看预估费用 |
| **线上支付** | 官方商品、教练教学、摄影跟拍 | 用户→平台→自动分账 | 在线支付后报名成功 |

### **3.2 线上支付的两种模式**

#### **模式A：平台官方收款（100%归平台）**
```
用户支付 ¥100
    ↓
进入平台账户（¥100全额）
    ↓
用于：官方团购、官方活动、赛事奖品等
```

**适用场景**：官方团购、官方活动、赛事报名费

---

#### **模式B：个人收费服务（平台自动分账）**
```
用户支付 ¥100
    ↓
进入平台账户
    ↓
自动分账：
├─ 服务提供者（教练/摄影师）：¥85 (85%)
└─ 平台佣金：¥15 (15%)
    ↓
服务提供者可提现到个人账户
```

**适用场景**：认证教练的付费教学、认证摄影师的付费跟拍

**对接现有功能**：
- ✅ 个人收款账户管理（教练/摄影师已有个人账户）
- ✅ 分账规则配置（85/15分成比例，系统自动计算）
- ✅ 提现功能（服务提供者可将收入提现到微信/支付宝/银行卡）

### **3.3 发布时费用设置界面**

**【创建活动 - 费用设置】**

```
费用类型：[单选]

○ 免费活动
  无需任何费用，纯社交组局

○ 线下AA
  ├─ 预估人均费用：¥_____ （选填，仅供参考）
  └─ 费用说明：例如"包含油费、过路费，现场AA结算"

● 线上支付
  ├─ 服务费用：¥_____ /人 （必填）
  ├─ 人数限制：最多___人 （必填）
  ├─ 退款规则：活动开始前24小时可全额退款（系统默认）
  └─ 分账说明：
      • 平台官方发起：100%归平台
      • 认证教练/摄影师发起：自动分账（你收85%，平台收15%）
```

**权限说明**：
- 普通用户：只能选择"免费"或"线下AA"
- 认证教练/摄影师：教学/摄影活动可选"线上支付"（85/15分账），其他活动只能"免费"或"线下AA"
- 俱乐部：约滑、教学、吃喝玩乐、摄影、赛事活动可选"线上支付"（85/15分账）
- 平台官方：全部活动可选"线上支付"（100%归平台或85/15分账）

## **四、 供给侧：动态发布器字段定义 (Dynamic Form Schema)**

### **【公共基础模块】(所有类型都有)**

* **标题：** 限30字。
* **封面图：** 多图上传/视频（最多9张）。
* **活动时间：** 开始时间-结束时间。
* **集合地点：** LBS定位（地图选点）。
* **活动标签：** \[多选标签\] 用于体现活动性质。
  * 官方活动可选：开板节 / 品牌试滑 / 嘉年华 / 官方认证
  * 俱乐部可选：俱乐部活动 / 年会 / 团建 / 内部赛
  * 通用标签：AA搭子 / 小团体 / 新手友好 / 高手进阶
* **费用类型：** \[单选\] 免费 / 线下AA / 线上支付。
  * *免费*：直接报名。
  * *线下AA*：填写"预估人均费用"（选填）。
  * *线上支付*：填写"服务费用"和"人数上限"（必填）。
* **人数限制：** \[数字\] 最多接受多少人报名（选填，线上支付时必填）。
* **活动说明：** \[富文本\] 详细描述活动安排、注意事项、日程表等。
  * 大型活动可在此编辑日程表：例如"09:00签到 → 10:00试滑开始 → 12:00午餐"
  * 可插入主办方Logo、品牌介绍等

## **五、 消费侧：前端展示与交互 (Client UI)**

### **5.1 列表页 (Feed) \- 卡片差异化**

**通用布局**：上图下文（或左图右文），但关键信息区针对不同活动类型差异化展示。

| 活动类型 | 卡片核心信息 | 关键视觉元素 |
| :---- | :---- | :---- |
| **约滑** | 水平要求 + 板型 + 时间地点 + 活动标签 | 徽章图标（单板/双板）+ 标签（如"开板节"） |
| **教学** | 教练资质 + 教学内容 + 价格 + 人数 | 认证标识 + 教练头像 |
| **拼车/出行** | 路线（出发地→目的地）+ 剩余座位数 | 车辆图标 + 路线箭头 |
| **拼房/住宿** | 性别限制（限男/限女）+ 房型 + 人均费用 | 高亮Icon（性别标识） |
| **吃喝玩乐** | 聚局主题 + 时间地点 + 活动标签 | 主题图标 + 标签（如"俱乐部年会"） |
| **摄影/跟拍** | 服务类型 + 设备 + 价格 + 样片 | 摄影师认证标识 + 样片缩略图 |
| **赛事/挑战** | 挑战目标 + 当前第一名成绩 + 奖品 | 奖杯图标 + 排名数字 + 主办方标识 |
| **官方团购** | 价格 + 折扣 + 拼团进度条 | 倒计时 + 省钱金额 |

**说明**：
- 官方活动、俱乐部活动通过**活动标签**和**发起方信息**体现，不需要单独的卡片样式
- 大型活动（如开板节）可选择使用**大通栏海报样式**（通过封面图实现）

### **5.2 详情页 (Detail) \- 通用结构**

**所有活动详情页包含以下通用模块**：

1. **Header区**：封面图/视频 + 活动类型标签
2. **核心信息区**：
   * 活动标题
   * 时间、地点、人数限制
   * 费用信息（免费/线下AA/线上支付）
3. **发起人信息区**：
   * 发起人头像、昵称、实名认证状态
   * （线上支付活动）平台官方标识
4. **活动详情区**：
   * 活动说明（富文本）
   * 垂直字段（如拼车的车辆信息、约滑的水平要求等）
5. **报名列表区**：已报名用户头像（最多显示9个，超出显示数字）

### **6.6 核心业务规则**

#### **支付规则**

1. **支付超时**：创建订单后15分钟未支付，订单自动取消释放名额
2. **库存管理**：线上支付活动必须设置人数上限，报满后不可报名
3. **支付渠道**：支持微信支付、支付宝支付
4. **资金流向**：所有资金进入平台账户（MVP不做个人分账）

#### **退款规则**

1. **自动退款**：活动开始前24小时以上申请，系统自动通过并退款
2. **不可退款**：活动开始前24小时内，无法申请退款
3. **活动取消**：发起方取消活动，所有订单自动全额退款
4. **退款渠道**：原路返回（微信退微信，支付宝退支付宝）

#### **赛事/挑战规则**

1. **参赛资格**：
   - 免费赛事：直接参赛
   - 付费赛事：支付报名费后参赛
   - 可设置参赛门槛（如水平要求、实名认证等）

2. **数据采集**：
   - 自动从App轨迹记录器获取数据
   - 仅统计使用本App记录的滑雪轨迹
   - 数据包括：里程、速度、滑行时间等

3. **排行榜更新**：
   - 每10分钟自动更新一次排名
   - 支持多维度排行：总榜、男子榜、女子榜、单板榜、双板榜
   - 显示实时排名和成绩

4. **赛事结束**：
   - 到达结束时间后，自动封榜
   - 公布最终排名和获奖名单
   - 推送通知给获奖用户

5. **作弊防范**：
   - 轨迹数据需经过系统验证（GPS轨迹合理性检查）
   - 异常数据将被标记或剔除

#### **通知规则**

**需要推送通知的场景**：
1. 支付成功 → "报名成功，请按时参加活动"
2. 退款成功 → "退款已到账，请注意查收"
3. 发起方取消活动 → "活动已取消，费用已退回"
4. 活动开始前1天 → "您报名的活动明天开始"
5. 订单即将超时 → "订单还有5分钟过期，请尽快支付"
6. 免费/AA活动报名成功 → "报名成功，可联系发起人了解详情"
7. **赛事参赛成功** → "参赛成功，开始记录你的滑雪轨迹吧"
8. **赛事排名变化** → "你在XX赛事中的排名上升至第X名"（可选推送）
9. **赛事即将结束** → "XX赛事还有24小时结束，加油冲刺"
10. **赛事结束获奖** → "恭喜！你在XX赛事中获得第X名"
