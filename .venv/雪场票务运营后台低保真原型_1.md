# 雪场票务运营后台 - 低保真文字原型

## 一、系统整体架构

```
自我游票务系统
       ↓ (API同步)
【模板中心】雪场模板 + 产品类型模板
       ↓
【票务库】自我游同步的原始票务
       ↓ (套用模板快速创建)
【产品库】可销售的商品
       ↓ (上架分发)
【渠道端】C端APP + B端分销商
       ↓ (交易履约)
【订单管理】支付/退款/核销
       ↓
【财务结算】供应商/分销商结算
```

## 三、核心业务流程

### 流程1：从票务到上架
```
1. 自我游每天凌晨2点全量同步票务到【票务库】

2. 运营人员登录后台：
   a. 首次使用：先配置【雪场模板】和【产品类型模板】（一劳永逸）
   b. 日常使用：从票务库选票务，套用模板，快速创建产品
   
3. 快速创建单个产品：
   票务库选票务 → 选择模板 → 微调标题/价格 → 上架
   耗时：< 2分钟
   
4. 批量创建产品：
   票务库批量选票务 → 选择模板 → 批量定价 → 批量上架
   耗时：< 3分钟（10个产品）
   
5. 产品上架后：
   - 自动同步到C端APP
   - 自动对B端授权的分销商可见
   - 库存每10分钟自动同步
```

---

### 流程2：用户购买到核销
```
C端用户流程：
1. 用户在APP选择产品、日期、数量
2. 下单支付
3. 后台实时检查库存（调自我游API）
4. 支付成功→自动向自我游下单锁库存
5. 获取核销码，推送给用户（短信+APP）
6. 用户到雪场扫码核销
7. 核销信息同步至自我游+雪场系统

B端分销流程：
1. 分销商推广产品给会员
2. 会员通过分销商链接/代理下单
3. 订单自动打上分销商标签
4. 后续流程同C端
5. 按周期结算佣金给分销商
```

---

### 流程3：退款处理
```
1. 用户在APP申请退款，填写原因

2. 退款申请进入后台【订单管理-退款中】

3. 运营人员审核：
   a. 系统自动匹配退改规则，计算退款金额
   b. 核对订单状态（未核销才能退）
   c. 决定同意/拒绝

4. 如果同意退款，系统自动执行：
   a. 取消我方订单
   b. 调用自我游API取消订单
   c. 释放库存
   d. 发起退款到用户原支付方式
   e. 更新订单状态为"已退款"
   f. 短信/APP通知用户

5. 如果拒绝退款：
   填写拒绝原因 → 通知用户

6. 退款记录进入【财务结算-退款记录】
```

---

### 流程4：库存实时同步
```
自动同步：
• 每10分钟定时同步所有在售产品库存
• 用户下单时，实时调用自我游API检查库存
• 用户支付成功后，立即锁定库存

预警机制：
• 库存<50张：站内信+短信提醒运营
• 库存<10张：自动下架产品
• 售罄：自动下架，运营可手动补货后重新上架

异常处理：
• 同步失败：自动重试3次
• 重试仍失败：记录日志，通知运营人工处理
```

---

### 流程5：模板更新同步
```
场景：雪场模板更新了交通指南

1. 运营人员编辑【万龙雪场】模板，更新交通指南

2. 保存时，系统检测到该模板被45个产品使用

3. 弹出提示：
   "该模板更新将影响45个产品，是否同步？"
   [自动同步全部] [手动选择] [暂不同步]

4. 如果选择自动同步：
   系统批量更新45个产品的交通指南内容
   
5. 如果产品已脱离模板：
   不受影响，保持独立内容

6. 同步完成后通知运营
```

## 结语

本运营后台通过**模板体系**大幅降低运营维护成本，通过**实时同步**保证库存准确性，通过**自动化流程**提高处理效率，通过**统一管理**实现C端B端双渠道运营。
