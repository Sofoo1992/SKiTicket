## 🎯 核心设计原则

### 1. 极简原则
- 产品名称：直接取票务库名称，不做复杂生成
- 模板配置：只保留必要字段（5-8个）
- 去掉复杂逻辑：无变量系统、无解析提取

### 2. 高效原则
- 单个产品：2分钟创建
- 批量创建：3分钟10个
- 批量维护：调价、上下架、修改渠道

### 3. 灵活原则
- 产品名称：初始使用票务库名称，后续可随时优化
- 详情页内容：初始使用模板，可随时编辑或脱离模板

### 4. 复用原则
- 雪场/酒店模板：配置一次，多个产品自动引用
- 模板更新：批量同步到关联产品

## 🔑 关键业务流程

### 票务库 → 产品库 → C端展示

```
自我游票务系统
    ↓ (每天同步)
票务库（日历库存）
    ↓ (选择票务)
应用模板（雪场+分类/酒店）
    ↓ (自动填充80%内容)
产品库
    ↓ (上架)
C端展示（雪哒APP）
    ↓ (用户购买)
订单管理（拆单、核销）
```

## 💡 关键技术点

1. **库存按日期**
   - 票务库存储日历库存表
   - 产品实时引用，不存储库存
   - 每10分钟从自我游同步

2. **连续购买拆单**
   - 1个主订单 + N个子订单
   - 每天独立核销
   - 支持部分退款

3. **模板自动引用**
   - 雪场/酒店信息自动同步
   - 商家信息全局统一
   - 模板更新，产品自动更新

4. **双重库存校验**
   - 前端实时查询（用户体验）
   - 后端下单校验（安全保证）
   - 支付后锁定库存
